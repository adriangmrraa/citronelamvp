Temas

Presupuesto y estructura de pago

  - Se propuso un presupuesto de ~$4,800, basado en una tarifa de $20/h (nivel junior) para mantener bajos los costos.
      - Cálculo: 4 h/día × 5 días/sem × $20/h = $400/sem → $1,600/mes × 3 meses = $4,800.
  - El pago es en ARS, convertido desde USD usando el tipo de cambio oficial promedio.
  - Se propone un plan en cuotas de 6 meses ($800/mes).
      - Justificación: Esta estructura genera confianza y reduce el riesgo de la inversión inicial del cliente.
      - Validación: Un cliente anterior (Marcelo) que comenzó con este plan luego duplicó su pago mensual para ampliar el proyecto.

Marco contractual

  - Se revisó un contrato de muestra para establecer una base clara del proyecto.
  - Cláusulas clave:
      - Alcance: Definido por un documento de "ingeniería de requerimientos".
      - Cronograma: ~3 meses, con un calendario diario detallado.
      - Propiedad intelectual (PI): El cliente obtiene derechos de uso tras el pago total; el desarrollador conserva el know-how.
      - Responsabilidad: El desarrollador no es responsable por fallas de servicios de terceros (p. ej., Cloudflare).
      - Rescisión: Aviso con 15 días; el cliente paga las horas trabajadas.

Camino a seguir del proyecto

  - Un presupuesto formal requiere un documento de "ingeniería de requerimientos" por parte del cliente para definir el alcance y las limitaciones del proyecto.
  - El equipo creará un grupo de WhatsApp para comunicación y retroalimentación directa.
